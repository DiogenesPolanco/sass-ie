What's going on here?

Building CSS mobile-first is the way forward, because blah blah blah progressive enhancement blah. Problem is, Internet Explorer prior to 9 ignore anything within media query blocks, leaving those browsers with mobile styles.

Not all of us get away with that, but thankfully Sass 2.3 (not yet released) can generate a seperate stylesheet with everything it needs to create a desktop look & feel.

_layout.scss

This file controls the page layout:

.page {
    @include respond-min(70em) {
        width: 70em;
        margin: 0 auto;
    }    
}
.content-margins {
    margin: 10px;

    @include respond-min(60em) {
        margin: 20px;
    }
}
.primary {
    @include respond-min(45em) {
        float: left;
        width: 70%;

        @include old-ie {
            // These hacks won't appear in the normal stylesheet
            display: inline;
            zoom: 1;
            whatever-ie-needs: hacks;
        }

        & .content-margins {
            margin-right: 0;
        }
    }
}
.secondary {
    @include respond-min(45em) {
        float: left;
        width: 30%;
    }
}
